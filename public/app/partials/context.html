  • Test-Client: UI where you can see a behaviour-simulation of all of those twenty (20) runners during competition, TIMER starts automatically. You can see as a log and as dinamyc sets the chip codes "detected" by each Timing Point.
  • API: A small set of available resources implemented on the backend which receives the timing information in real-time.
